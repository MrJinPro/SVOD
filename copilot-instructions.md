# Инструкции для Copilot (локальная среда)

## Важно

- **Не запускать проект локально** (никакие `run_task`, `uvicorn`, `npm dev/build/preview` и т.п.).
- **На этой локальной машине нет базы данных** — не пытаться проверять функционал через локальный backend/SQLite/Postgres.

## Процесс работы

- Вносим изменения в код.
- Проверки выполнения/запуска делает пользователь **после push** в репозиторий.
- Если нужно подтвердить работу изменений — просим пользователя запустить у себя/на ВМ и прислать результат/логи.
